Feature Type Designer

Introduction

Designing method of feature types is introduced in this page. As already mentioned at Modeler page, GFM specifies that feature type may define more than one attribute types and operation types as its properties. A feature type may inherit properties of a parent feature type. And a feature type associates with other feature types through association types. For example, an association from "Road" to "Bridge" may be represented by "RtoB" association. Road links RtoB as a from-feature type, and Bridge is linked by RtoB as a to-feature type.

A feature type is an abstract type or a concrete type. Abstract type can not create instances. While, concrete type may create instances. For example, in the land-use planning, zoning is used to lead better environment. In Japan, zone types for land-use planning in a city are separated into home district, business district, and industry district. These districts inherit properties of "use district". Use district has attributes for the land use restriction such as classification of building built in the district, building height restriction, plot (floor-space) ratio, building-to-land ratio and so on. These attributes are inherited by home, business and industry districts.

An abstract type cannot be a child of a concrete type as it is not concrete.

Feature type designer is used to define and edit the definition of feature types. Attribute types are defined by Attribute Type Designer page and operation types are defined by Operation Type Designer. They can be called from Feature Type Designer.

Feature Type Designer

Figure 1. Page for feature type designing

Fields

Feature Type (editable)
Feature type name is keyed-in. The first letter shall be uppercase (e.g., Building). If the name is a combination of words, the first character shall be uppercase and words are connected (e.g., CityBoundary). A name of feature types shall be unique.

definition (editable)
A definition of the feature type is described here.

attributes
Declared attribute types are listed here. The declaration is performed in Attribute Type Designer page.

inherited attributes
Attribute types inherited from the parent type is listed here.

operations
Declared operation types are listed here. The declaration is performed in Operation Type Designer page.

inherited operations
Operation types inherited from the parent type is listed here.

proxy
The name of proxy type is shown here.Proxy is a representative attribute of a feature. The concrete feature type shall have a proxy. Currently, gittok allows geometry or character string may be a proxy. Proxy is mainly used for the creation of association instances. Association indicates which feature links to other feature(s). To define the link between features, the user selects features through proxies displayed on the map and/or the list. Editer page (Acquisition of feature instances) explains how to use proxy in detail.

links
the list of association types the feature type links is listed here. The feature type becomes a from-feature type.

inherited linkes
Inherited association types the feature type links are listed here.

linkedBy
the list of association types the feature type being linked by is listed here. The feature type becomes a to-feature type.

inherited linkedBy
Inherited association types the feature type being linked by are listed here.

feature types
Declared feature types is listed here.

parent
The name of parent type is shown here by clicking "as parent" button after the selection of the feature type on "feature types" list.

children
Child feature types if the feature type is a parent are listed here.

Button

isAbstract
The feature type becomes "abstract" by clicking the check box.

add
The declared feature type is added by clicking this button.

edit
The feature type can be edited by clicking thiss button after the selection of a feature type.

update
The feature type is replaced by the edited type by clicking this button.

delete
The feature type selected on the list is deleted by clicking this button. The deletion is impossible, if the feature type has child types.

asParent
the selected feature type become a parent type of the feature type edited now by clicking this button.

cancel
The parent type is canceled by clicking this button.

clear
All parameters of the feature type currently edited eliminated by clicking this button. The new definition become possible by clearing parameters.

edit attributes
Attribute type declaration become possible on Attribute Type Designer by clicking this button.

edit operations
Operation type declaration become possible on Operation Type Designer by clicking this button.
